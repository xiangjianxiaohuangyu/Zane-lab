## 项目概述
Element Loop 是一款像素类俯视角 2D 肉鸽游戏。本项目将元素作为世界状态变量，将元素反应视为规则推演过程。通过操纵环境、诱导反应、修改规则，玩家参与一个持续演化的元素系统。项目旨在通过像素级的地形破坏与深度的元素反应系统，重新定义“自动攻击”类 Roguelike 游戏。

游戏强调：
- 元素不只是伤害类型，而是**可传播、可叠加、可演化**的状态
- 反应不是固定公式，而是**链式、多阶段的系统行为** 
- 构筑的核心不是装备或技能，而是 **规则与约束本身**

## 核心设计理念
1. 元素即世界（Element as World State）

地图中的每一个区域都携带元素状态，例如火、水、雷、风、冰等。元素具有强度、持续时间与扩散规则，并随时间与交互发生变化。世界本身持续进行元素运算，而非仅在技能命中时触发效果。

2. 反应即推演（Reaction as Inference）

元素反应并非简单的“二元触发”，而是多步、可传播、可被中断或放大的过程。例如：

火 + 水 → 蒸汽

蒸汽 + 雷 → 导电云

导电云 + 风 → 雷暴区域

玩家的策略重点在于引导反应路径，而非追求单次高伤害。

3. 规则即策略（Rules is Strategy）

游戏不追求单纯的数值叠加，而是强调物理属性的改变。作为 Roguelike 的核心随机性来源，游戏中的构筑并非简单数值提升，而是对底层规则的修改，例如：

元素是否可以残留

反应是否产生次级元素

扩散方向是否可控

反应是否稳定或失控

规则本身成为可收集、可组合、可污染的资源。

## 创新点总结
- 系统驱动而非角色驱动：玩家操纵的是规则与环境
- 链式元素反应：反应具有传播性与时间维度
- 规则型构筑：改变“世界如何运作”，而非“角色有多强”
- 高可复玩性：每一局的元素法则都可能不同
- 实验失败也是体验的一部分：失控、污染、灾变是合法结果
